# 演奏表现力实验室

这是一个网页版钢琴演奏表现力分析 Beta。当前版本把“真实上传分析”和“Demo 示例数据”明确隔离：默认空状态不显示任何样例评分，只有上传可解析 MIDI 或主动打开 Demo 后才进入报告。

## 定位

- 产业：面向音乐学院学生、钢琴教师、独立音乐人和排练指导的练习/教学工作台。
- 学术：提供 timing、dynamics、articulation、phrasing、stability、style distance 的可解释分析界面。
- 科研转化：后续可扩展 MIDI、MusicXML、CSV、音频对齐标注等数据导出能力。

## 量化原则

本项目不把“艺术价值”简化成一个绝对分数，而是描述可观察的演奏选择：

- 速度处理：局部速度塑形、起音偏差、rubato 曲线。
- 力度塑形：实际力度或响度在乐句中的轮廓。
- 触键控制：实际时值与谱面/参考时值的比例。
- 乐句连贯性：乐句边界、句尾释放、力度拱形和内部稳定性。
- 钢琴多声部：和弦同步、声部突出、织体密度、句法气口。
- 演奏稳定性：演奏选择是否可控，而不是随机波动。
- 风格距离：当前演奏与专家参考版本的相似程度。

每个评分都对应图表证据，应该能追溯到乐句、拍点或音符级偏差。

## 当前范围

已实现：

- 钢琴单乐器试行版。
- Demo 示例钢琴乐句数据，界面明确标识为示例，不代表用户演奏。
- 音符级表现力指标计算函数。
- React 网页工作台。
- 无外部图表库的 SVG 可视化。
- MIDI 文件本地解析试行版：上传 `.mid/.midi` 后在浏览器内读取音符，不发送到外部服务。
- 钢琴多声部 MIDI 分析：支持正常钢琴片段里的多声部、和弦、织体密度和句法气口判断。
- 保留 16 音参考对齐模块：作为后续弦乐、管乐或指定谱面片段开发的准备，不再作为钢琴默认上传路径。
- 音频文件本地解析：仅显示录音质量、力度包络和起音能量，不生成音符级表现力评分。
- 文件安全限制：钢琴 MIDI 最大 1MB、最多 2000 个有效音符；音频最大 25MB、最长 3 分钟、解码超时 15 秒。

暂不包含：

- 任意曲目的自动谱面识别或复杂 MIDI 对齐。
- 音频转谱、音频到音符级对齐、音频级教师评分。
- 非钢琴乐器。
- 专家标注流程。
- 与教师评分的统计验证。

## 钢琴 MIDI 上传说明

当前钢琴上传功能用于分析正常钢琴 MIDI 片段，不再限制为单声部旋律。系统会把近似同时发生的音符归为和弦事件，并计算：

- 和弦同步：多音同时下键的紧密度。
- 声部突出：旋律或高声部是否从和声中浮现。
- 句法气口：句尾释放、停顿与再进入位置。
- 时间流动：整体推进是否平稳可控。
- 织体控制：多声部密度变化是否稳定。

当前钢琴模式不需要专家参考轨，也不要求音高顺序匹配某个 16 音片段。若需要“学生版本 vs 专家版本”的音符级对齐，后续应接入谱面 MIDI / MusicXML 或教师标注的参考版本。

## 参考对齐模块的保留用途

`src/lib/midiUpload.ts` 和 `src/lib/performanceMetrics.ts` 中的 16 音参考对齐体系仍保留。它更适合后续开发：

- 弦乐、管乐、声乐等单旋律或少声部乐器。
- 指定练习片段的学生版 vs 专家版比较。
- 有谱面 MIDI / MusicXML / 教师标注参考轨的音符级偏差分析。

## 音频上传说明

音频上传当前只做本地音频级分析，包括时长、采样率、声道、力度包络、起音能量、静音比例和录音风险提示。它不生成“表现力评分”“教师评价”或音符级偏差报告。要获得乐句、触键和音符偏差分析，需要上传可对齐 MIDI，或后续接入谱面/音频转录能力。

当前音频限制：最大 25MB、最长 3 分钟、解码超时 15 秒。超过限制会直接进入不可分析状态，避免浏览器卡死。

## 环境要求

- Node.js >= 22.12.0
- npm ci 用于发布/CI 安装

## 运行

```bash
npm install
npm run dev -- --port 4301 --strictPort
```

打开：

```text
http://localhost:4301/
```

## 发布门禁

上线前必须通过：

```bash
npm ci
npm run test
npm run build
npm audit
```

当前 `npm run build` 已包含单元测试、TypeScript 编译和 Vite 生产构建。`npm audit` 必须为 0 个 high / critical 漏洞。

默认 `dev` 和 `preview` 只监听 `127.0.0.1`。如需局域网调试，使用 `npm run dev:lan` 或 `npm run preview:lan`。

## 隐私边界

- MIDI 与音频文件均在浏览器本地解析。
- 当前版本没有账号系统、没有后端上传、没有云端保存。
- 若未来接入后端音频转录、教师批改或历史记录，必须在上传前向用户说明文件用途、保存周期和删除方式。

## 上线验收清单

- 默认首页不得显示 Demo 分数、图表或练习建议。
- Demo 报告必须显示“Demo 示例数据，不代表你的演奏”。
- 钢琴 MIDI 上传可分析多声部、和弦、织体密度和气口，不得退回单声部限制。
- 音频上传只显示音频级分析，不生成表现力评分。
- 错误状态不得残留上一次成功结果。
- 移动端不得出现横向滚动。
